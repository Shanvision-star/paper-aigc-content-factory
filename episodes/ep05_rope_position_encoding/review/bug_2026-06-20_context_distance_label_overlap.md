# EP05 Context 距离标尺文字压线记录

## 现象

用户截图指出 context 场景里“相对位移 n - m”文字与横向距离线段发生重叠，影响专业观感。该画面用于说明长上下文下需要稳定区分位置 m、位置 n 和相对位移 n - m。

## 影响

- 观众会把重叠误读成图表错误，而不是注意到“相对位移”这个主机制点。
- 该帧是公开证据页后的工程解释页，若标尺不清楚，会削弱“长上下文让位置建模变成核心工程问题”的 Big Idea。
- 文字和线段叠在一起也违反 EP05 动画约束：公式、图表、字幕和标注不得互相遮挡。

## 根因

`build_ep05_final_mp4.ps1` 中 `.context-scale-brace` 同时承担括线容器和文字标签。原布局把文字放在括线框内部，并让括线、横向 track 和文字处于接近的垂直区域；在 1080x1920 渲染后，粗体中文基线和线段视觉上压在一起。

## 修复方案

- 将文字拆成独立 `.context-scale-brace-label`，不再放在 `.context-scale-brace` 内。
- 将括线 `.context-scale-brace` 下移，只负责绘制相对位移范围。
- 增大 track 高度和标题到 track 的间距，形成“标签在上、括线在中、横线在下”的三层结构。

## 验证方式

- 使用 PowerShell parser 检查 `build_ep05_final_mp4.ps1` 模板语法。
- 重跑 EP05 MP4 或至少重跑 HyperFrames 后抽取 context 场景关键帧，确认“相对位移 n - m”在横线之上，且不与括线、点位或底部动态字幕重叠。

## 预防约束

- 标尺类图形不得让同一个 DOM 节点同时承担文字和线条语义。
- 所有线段标注采用独立 label；label、brace、track 至少保留 8px 以上视觉间距。
- 广告学审核时该帧的 visual hero 是“m 到 n 的相对位移标尺”，任何重叠都会削弱 proof object，必须作为 P1 视觉问题处理。
