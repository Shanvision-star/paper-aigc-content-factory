# EP05 Outro “与”字渲染异常记录

## 现象

用户截图指出最终 outro 页大字“位置进入 Attention，下一步就是 KV cache 与推理速度。”里的“与”观感异常，被标注为“与字乱码”。同一页标题和底部可见区域也存在“KV cache 与推理速度”的同类混排表达。

## 影响

- 影响最终页的专业观感，容易被理解为编码错误或画面乱码。
- 问题出现在视频收束页的大字号核心文案，会削弱 EP05 结尾对下一集主题的可信度。
- 若只改中心大字而不处理标题、字幕烧录或分镜合同，后续重跑仍可能把同一表达带回画面。

## 根因假设

当前源码中的文本本身不是错字，也不是文件编码损坏；`build_ep05_final_mp4.ps1` 中可见文案确实写的是“KV cache 与推理速度”。异常更可能来自大字号 CJK + Latin 混排时，中文“与”在当前字体链路里触发 fallback glyph、字重匹配或行内渲染差异，导致单字观感像乱码。

## 修复方案

- 将 outro 场景标题从“下一集：KV cache 与推理速度”改为“下一集：KV cache 和推理速度”。
- 将 outro 中心大字从“位置进入 Attention，下一步就是 KV cache 与推理速度。”改为“位置进入 Attention，下一步就是 KV cache 和推理速度。”。
- 在字幕读取阶段对可见烧录字幕做精确短语归一化，把“KV cache 与推理速度”显示为“KV cache 和推理速度”，不修改口播含义、不改音频、不重生成 MP4。
- 同步 `video_script/FRAME.md` 的过渡约束，防止未来按旧分镜合同回退。

## 验证方式

- 静态检索 `build_ep05_final_mp4.ps1`，确认最终可见中心大字和 outro 标题不再包含“KV cache 与推理速度”。
- 静态检索 `FRAME.md`，确认复用约束已改为“KV cache 和推理速度”，并记录 Original / Revised / Reason / Impact。
- 使用 PowerShell parser 对 `build_ep05_final_mp4.ps1` 做语法检查。
- 本次不生成最终 MP4，不触碰音频，不改 MATLAB 资产；主线程如需画面级确认，应重跑视频或至少重跑 outro 截图审核。

## 预防约束

- 大字号可见文案中遇到 `English term + 与 + 中文术语` 的 CJK/Latin 混排，优先改写为“和”或顿号并列，避免单字 fallback 风险。
- 可见画面不得暴露制作提示词、QA、Hook、视觉焦点、教学边界、读作等内部词。
- 动态字幕仍是固定字幕槽按时间替换，不使用文字缩放动画。
- 每页 EP05 顶部标题继续统一为“EP05 · 为什么现代大模型都绕不开 RoPE？”。
