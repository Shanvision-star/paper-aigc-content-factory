# EP05 RoPE 口播稿 V3 216s 待审核

## 版本目标

- 总时长目标：约 216 秒，最终成片不得低于 180 秒。
- 节奏目标：普通观众能先看懂图，再听懂公式含义；复杂公式段不抢读。
- 动画目标：MATLAB 动画至少承担三次解释任务：整体对照、Q/K 旋转、`n - m` 公式结果。
- 技术边界：不把 RoPE 写成所有模型必用；不把 ChatGPT 当作 RoPE 实现证据；不说“点积后只剩 n - m”。

## 一句话 Big Idea

原始 Positional Encoding 把位置加到输入里；RoPE 把位置转进 Q 和 K 的匹配关系，让 Attention 在比较时更直接看到相对位移 `n - m`。

- `00:23-00:50` 给 Annotated Transformer 代码示例里的 `x = x + pe` 足够停留时间，不让观众误解成拼接或新增 token。

- `02:02-02:31` 给 MATLAB RoPE true-meaning 动画至少 29 秒，允许重复播放、局部放大和公式停顿。
- `02:50-03:00` 只做公开模型边界，不扩展未经公开的位置编码细节；删除 ChatGPT 口播段，避免把专有产品语境做成独立记忆点。

## TTS spoken_text 合同

| source_text / 画面显示 | spoken_text / 音频读法 |
| --- | --- |
| `mθ_i` | m 乘 theta 下标 i |
| `nθ_i` | n 乘 theta 下标 i |
| `θ_i` | theta 下标 i，也就是第 i 个二维块的旋转频率 |
| `δ_i` | delta 下标 i，也就是第 i 个二维块里的相对角度 |
| `n - m` | 画面公式显示；口播只解释成“相对位移 / 相隔多远”，避免重复机械朗读 |
| `d_model` | d model |
| `x = x + pe` | x 等于 x 加 P E |
